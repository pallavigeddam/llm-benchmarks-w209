04 —Why Benchmarks Break: The Missing Context

Two forces degrade every static benchmark — saturation and contamination. Our hypotheses capture saturation. Here's what makes it structurally worse.

The contamination problem: 29% of MMLU was compromised before the race started

Independent audits found 29.1% of MMLU questions appear in LLM training data (NAACL 2024). Models tested on a clean parallel dataset dropped 13 points on GSM8K (Scale AI, 2024). Some of the apparent "improvement" reflects memorization, not reasoning. Static benchmarks lose ranking signal in under two years on average.

05 —What You Should Actually Use Instead

If MMLU is saturated, cost doesn't signal quality, and rank inversions are common at the frontier — here is how practitioners should actually evaluate models.

Rule
Never trust a single benchmark. Triangulate across three types: a static academic eval (MMLU-Pro, GPQA-Diamond), a human-preference arena (LM Arena ELO — the metric that can't be gamed), and an agentic task suite (SWE-bench, GAIA). Agreement across all three is the signal.
LAYER 1 · STATIC

Academic benchmark

MMLU-Pro / GPQA-Diamond — use instead of plain MMLU. 10-choice format forces reasoning over recall. Longer signal runway than MMLU.

LAYER 2 · GROUND TRUTH

Arena ELO — the ungameable metric

LM Arena ELO — real humans, blind A/B preference voting. No answer key to memorize. No contamination possible. This is the ground truth in all three hypotheses.

LAYER 3 · AGENTIC

Task-completion benchmark

SWE-bench / GAIA / ARC-AGI-2 — real software engineering tasks. Tests deployment performance, not training data memorization.
